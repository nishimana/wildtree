# 設計書: WildTree v2 — ツリーエディタ（フェーズ1）

## 1. 概要・目的

Dynamic Prompts のワイルドカード YAML ファイル群（271ファイル、43,000キー定義）を、エントリポイントからツリーとして正しく展開表示し、その場で編集できるデスクトップアプリケーション。

### プロトタイプからの学び

v1 プロトタイプで以下の問題が判明した：
- トップレベルに大量のノードが並び、ツリーのルートが見つけられない
- クロスファイル参照（`__cards/パス/キー__`）が解決できず、ツリーがほぼ展開されない
- エントリポイント選択のプルダウンが使い物にならない（全キーが並ぶ）
- リテラルが非表示のため、キー定義の中身がわからない

### v2 のゴール

1. **トップツリーの自動検出**: 他から参照されないルートノード（`メイン`、`メインNP`）を自動で見つけて表示
2. **クロスファイル参照の完全解決**: 271ファイルを横断して `__cards/パス/キー__` を正しく辿る
3. **チェックボックスで有効/無効切り替え**: YAML コメント行のon/offをUIで操作
4. **キー名によるパス自動解決**: 長いパスを手書きせず、キー名だけで参照を指定
5. **リテラル表示**: プロンプトタグ等のリテラル値もツリーに表示

## 2. 用語定義

| 用語 | 意味 |
|---|---|
| キー定義 | YAML ファイルのトップレベルキーとその値行群 |
| 通常参照 | `__cards/パス/キー名__` — クロスファイル参照 |
| 動的参照 | `__{__変数__}サフィックス__` — 変数展開で参照先を組み立てる |
| トップツリー | 他のどのキー定義からも参照されていないキー = エントリポイント |
| 空定義 | `"{}"` — 未使用スロットを空にする Dynamic Prompts の慣習 |
| コメント行 | `# - __xxx__` — 無効化された値行。UIではチェックボックスOFFで表示 |

## 3. 機能仕様

### 3.1 トップツリーの自動検出・表示

起動時に cards ディレクトリを全スキャンし：
1. 全キー定義を収集
2. 全参照先を収集
3. 「参照されていないキー」= トップツリー候補を特定
4. トップツリー候補をリストで表示し、選択するとツリーが展開される

**想定されるトップツリー**: `メイン`、`メインNP` など少数

### 3.2 ツリー表示

選択されたトップツリーから参照を再帰的に辿り、ツリーを展開する。

```
メイン
├── [ref] デフォルト
│   ├── [ref] シネマシャドウ
│   │   └── (cinematic_shadow:1.1)           ← リテラル
│   └── [ref] バックライト
│       ├── (backlighting:1.2)               ← リテラル（選択肢1）
│       ├── backlighting                     ← リテラル（選択肢2）
│       └── (backlighting:0.8)               ← リテラル（選択肢3）
├── [ref] アングル
│   └── ...
└── [ref] シーンまとめ
    ├── ☑ [ref] 朝田詩乃シーンまとめ ...     ← 有効（非コメント行）
    ├── ☐ [ref] 白洲アズサシーンまとめ ...    ← 無効（コメント行）
    └── ☐ [ref] 結城明日奈シーンまとめ ...    ← 無効（コメント行）
```

#### 表示ルール
- **参照ノード**: キー名を表示、子ノードとして参照先を再帰展開
- **リテラル**: プロンプトタグ等の値をそのまま表示（リーフノード）
- **コメント行**: チェックボックス付きで表示（☐ = コメント = 無効）
- **空定義 `"{}"`**: 「(空)」等で表示
- **循環参照**: マークして打ち切り
- **未解決参照**: 赤字等で壊れた参照として表示

### 3.3 編集機能

#### コメント切り替え（チェックボックス）
- チェックを入れる → YAML の `#` を除去して値行を有効化
- チェックを外す → YAML の値行を `#` でコメントアウト
- 変更は YAML ファイルに即時反映
- ツリーが即座に再構築される

#### キー名による参照の自動解決
- 新しい参照を追加するとき、キー名を入力するだけでフルパスを自動解決
- 重複キー名がある場合は候補をリスト表示して選択
- 重複キー名の検知・警告機能

### 3.4 動的参照への対応（設計考慮）

`__{__cards/キャラキー__}NP__` のような動的参照：
- ツリー上では変数部分を展開して「この変数が解決されると、こうなる」を表示
- 例: `キャラキー` の現在有効な値が `朝田詩乃` なら → `朝田詩乃NP` として展開

**初期実装**: 動的参照は「動的参照ノード」として表示し、内部の `__変数__` を子ノードとして展開。完全な解決は後続で実装。

### 4.2 データモデル (`core/models.py`)

```python
@dataclass
class KeyDefinition:
    name: str                   # キー名
    file_path: Path             # 定義元ファイル
    line_number: int            # ファイル内の行番号（編集時に使用）
    values: list[ValueEntry]    # 値行のリスト

@dataclass
class ValueEntry:
    raw_text: str               # 生の値テキスト
    is_commented: bool          # コメント行かどうか
    line_number: int            # ファイル内の行番号
    refs: list[WildcardRef]     # この値行内の参照
    literals: list[str]         # この値行内のリテラル部分

@dataclass
class WildcardRef:
    name: str                   # 参照キー名（パス除去後）
    full_path: str              # 参照フルパス（cards/xxx/yyy）
    raw: str                    # 元テキスト（__cards/xxx/yyy__）
    is_dynamic: bool            # 動的参照かどうか
    inner_refs: list['WildcardRef']  # 動的参照内の変数参照

@dataclass
class TreeNode:
    name: str                   # 表示名
    key_def: KeyDefinition | None  # 対応するキー定義（リテラルノードはNone）
    node_type: NodeType         # ref / literal / dynamic / unresolved / circular
    children: list['TreeNode']
    value_entry: ValueEntry | None  # 対応する値行（チェックボックス用）

class NodeType(Enum):
    REF = "ref"                 # 通常参照（解決済み）
    LITERAL = "literal"         # リテラル値
    DYNAMIC = "dynamic"         # 動的参照
    UNRESOLVED = "unresolved"   # 未解決参照
    CIRCULAR = "circular"       # 循環参照
    EMPTY = "empty"             # 空定義 "{}"
```

### 4.3 パーサーパイプライン

v1 と同じ5段階だが、コメント行の保持とリテラル抽出が追加される。

| Stage | モジュール | 入力 | 出力 | v1 からの変更点 |
|---|---|---|---|---|
| 1 | `scanner.py` | cards ディレクトリ | YAML ファイルリスト | 変更なし |
| 2 | `parser.py` | YAML ファイル | KeyDefinition リスト | **コメント行も保持**（`is_commented=True`） |
| 3 | `parser.py` | 値行テキスト | WildcardRef + リテラル | **リテラル部分も抽出** |
| 4 | `resolver.py` | 参照名 + レジストリ | KeyDefinition | **動的参照の構造解析を追加** |
| 5 | `tree_builder.py` | エントリキー + Resolver | TreeNode ツリー | **リテラルノード・コメントノードを含む** |

### 4.4 トップツリー検出 (`core/top_tree.py`)

```python
def find_top_trees(registry: dict[str, list[KeyDefinition]]) -> list[str]:
    """他のどのキー定義からも参照されていないキー名を返す"""
    all_keys = set(registry.keys())
    referenced_keys = set()
    for key_defs in registry.values():
        for key_def in key_defs:
            for value in key_def.values:
                for ref in value.refs:
                    referenced_keys.add(ref.name)
    return sorted(all_keys - referenced_keys)
```

## 5. 参照解決アルゴリズム（v2）

### 通常参照

v1 と同じ。フルパス解決 + 短縮形解決。

### 動的参照

`__{__cards/キャラキー__}NP__` の場合：

1. 外側の `__...__` を検出
2. 内側の `{__cards/キャラキー__}` を検出 → 動的参照としてマーク
3. `キャラキー` の現在有効な値（コメントでない値）を取得
4. 変数を展開して最終的な参照名を組み立て
5. 組み立てた参照名で通常の名前解決を実行

**初期実装では**: 動的参照を検出し、内部の変数参照を子ノードとして表示するまで。完全な展開は後続。

### 重複キー検出

```python
def find_duplicate_keys(registry: dict[str, list[KeyDefinition]]) -> dict[str, list[KeyDefinition]]:
    """同名キーが複数ファイルに存在するケースを返す"""
    return {name: defs for name, defs in registry.items() if len(defs) > 1}
```

## 6. パフォーマンス考慮

- **43,000キー定義**: 初回スキャンに数秒かかる可能性がある → プログレスバー表示
- **ツリー構築**: 遅延展開（ノードを展開するときに子ノードを構築）で初期表示を高速化
- **編集後の再構築**: 変更されたファイルのみ再パースし、差分でレジストリを更新
