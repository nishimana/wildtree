# WildTree ビジョンメモ (2026-03-19)

## コアビジョン
- wildcard構文をツリーとして正しく表示し、その場で編集できるツール
- 現在のプロトタイプは出発点であり、設計は白紙で考え直す

## 実データから見えた構造

### スケール
- 271個のYAMLファイル、作品ごとにディレクトリ分離
- キー定義: 約43,000個
- エントリポイント: `__メイン__`（プロンプト用）と `__メインNP__`（ネガティブプロンプト用）

### 参照の流れ（メイン → 末端の例）
```
メイン
  → デフォルト, アングル, ライティング, シーンまとめ
    → シーンまとめ → 朝田詩乃シーンまとめ, 朝田詩乃メイン絵柄lora, 朝田詩乃お目々lora
      → 朝田詩乃 → タイトルプロンプト, キャラlora, 体格, 髪型, 目, 特徴, 性格ベース
        → 各定義（リテラル or さらに参照）
```

### 参照の種類
1. **通常参照**: `__cards/パス/キー名__` — クロスファイルの参照
2. **動的参照**: `__{__cards/キャラキー__}NP__` — 変数展開で参照先を組み立てる
3. **空定義**: `"{}"` — 未使用スロットを空にする慣習

### 現在のワークフロー上の痛点
1. キャラ切り替え = コメントのon/off（手動）
2. `__cards/作品名/CHキャラ名/キー名__` の長パスを手書き（ミスしやすい）
3. コメントで切り替えるキーが大量にある

## 設計方針（決定済み）

### フェーズ1: ツリービューア + エディタ（最優先）

#### ナビゲーション
- トップツリー（他から参照されないルートノード）を自動検出して起動時に表示
- そこから展開してツリーを辿る

#### 表示
- リテラルも表示する
- ツリー構造の正しい表示が最重要機能

#### 編集
- YAML直接編集方式（即時反映）。問題が出たら保存時出力に切り替え可
- コメント行はチェックボックスでON/OFF切り替え
- 排他/複数選択の判別はユーザー判断に任せる（どちらもチェックボックス）

#### 階層参照
- キー名で参照を指定、パスはアプリが自動解決
- キー定義の重複を検知して警告

#### 参照解決
- まず通常参照（`__cards/パス/キー__`）のクロスファイル解決を実装
- 動的参照（`__{__変数__}サフィックス__`）も対応する（設計段階から考慮）

### フェーズ2: チャット統合（後回し）
- Ply のチャット基盤（FastAPI + claude-agent-sdk）と連携
- WildTree 用の「場」を作り、project_path でワイルドカードディレクトリを指定
- Claude が YAML を直接読み書きできるようにする
- スキル統合も検討（/sd-dynamic-prompts-wildcards 等）
- **前提: Ply が安定稼働してから**
